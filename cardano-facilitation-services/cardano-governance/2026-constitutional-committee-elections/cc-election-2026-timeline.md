# CC Election 2026 - Timeline

{% hint style="warning" %}
As the process progresses some dates might get updated to accommodate community needs
{% endhint %}

{% stepper %}
{% step %}
#### Candidate Registration

**May 8th,** _9.45PM UTC_  **to June 21st,** _9.45PM UTC_

{% hint style="info" %}
During this phase, candidates may formally submit their applications.

* Registration is open to Individuals, Organisations, and Consortia
* All required fields must be completed
* Cold credentials **must** be provided and will be verified
{% endhint %}

Candidate profiles become visible once approved and can be edited until the voting phase begins.
{% endstep %}

{% step %}
#### Voting (Hydra Platform)

**June 28th**, _9.45PM UTC_  to **July 23rd**, _9.45PM UTC_

{% hint style="info" %}
During this phase:

* DReps cast stake-weighted votes
* Each DRep may vote for up to three candidates
* Voting takes place on Intersect’s Hydra-based voting platform
{% endhint %}
{% endstep %}

{% step %}
#### Voting Power Snapshot

**July 23rd**, _9.45PM UTC_

A snapshot of delegated stake is taken at the end of the voting period.

This snapshot determines the final voting weight used to calculate results.
{% endstep %}

{% step %}
#### Voting Audit

**July 23rd** to **July 26th**

An independent audit is conducted to verify the integrity and accuracy of the voting results.
{% endstep %}

{% step %}
#### Final Results

**July 26th,** _12PM UTC_

Audited results are published.

Candidates are ranked based on total stake-weighted support.
{% endstep %}

{% step %}
### Update Committee Governance Action – Preparation

**Late July 2026**

During this phase:

* Final candidate credentials are prepared
* The elected candidates are consolidated
* The Update Committee governance action is assembled
{% endstep %}

{% step %}
#### Update Committee Governance Action – Submission

**August 1st 2026**

Intersect submits the Update Committee governance action on-chain.

This action includes the elected candidates and initiates the formal ratification process.
{% endstep %}
{% endstepper %}
